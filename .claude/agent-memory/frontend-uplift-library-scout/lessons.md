# Library Scout Lessons

- **qtawesome cold-boot cost (2026-05-20):** qtawesome caches icon fonts on first use; on cold boot (no OS font cache), parsing and registering the icon font files with Qt's font engine adds ~150–200ms to app startup time. This is a one-time per-launch cost, not per-icon render. Acceptable for research desktop tools; cite in adoption guidance and consider lazy-importing qtawesome only when the first icon is requested rather than at module import time.
- **PySide6-Addons bundles QtWebEngineWidgets at zero marginal cost (2026-05-20):** `QtWebEngineWidgets` is part of `PySide6-Addons`, which is a required sub-dependency of the top-level `PySide6` meta-package. Any app already pinning `PySide6>=6.6` has `QtWebEngineWidgets` installed — importing it costs no additional disk space or pip dependency. Always check this before recommending matplotlib mathtext as the "lighter" alternative for rendered-equation tooltips.
- **pyqtdarktheme original is stale — always recommend the fork (2026-05-20):** The original `pyqtdarktheme` package (PyPI: `pyqtdarktheme`) last released v2.1.0 on 2022-12-25 and is unmaintained for newer Python versions. The active fork `pyqtdarktheme-fork` released v2.3.6 on 2026-03-04 and supports Python through 3.14. Never recommend the original package; always recommend `pyqtdarktheme-fork` and pin `>=2.3`.
- **pyvista <0.49 pin is correct but will need widening (2026-05-20):** PyPI confirms 0.48.4 is the latest stable pyvista release as of 2026-05-20. No 0.49 exists. The current `pyvista<0.49` pin is not blocking anything today, but it will need to be widened to `<0.50` (or per the next release) when 0.49 ships. Track PyVista releases before each panel-refresh sprint.
- **PySide6-QtAds has a hard version pin on PySide6-Essentials (2026-05-20):** The `PySide6-QtAds` PyPI package requires `PySide6-Essentials==6.11.0` (exact pin at time of survey). This creates a version-lock risk when the app's `PySide6>=6.6,<7` pin is on a different minor. Verify the exact pin before adopting QtAds, and consider whether the QtAds version can be upgraded to match the app's PySide6 version.
- **pymeshfix license is AGPL-3.0, not GPL-2.0+ (2026-05-21):** PyPI metadata for pymeshfix 0.18.1 reports "GNU Affero General Public License v3" — stricter than the source-registry.md entry which says "GPL-2.0+". AGPL-3.0 is a MAJOR redistribution blocker; do not import pymeshfix into any redistributable binary under any open-distribution model. Always re-verify the PyPI license field rather than relying on the registry entry.
- **superqt's QCollapsible uses QPropertyAnimation internally (2026-05-21):** superqt's `QCollapsible` widget drives its expand/collapse animation via Qt's built-in `QPropertyAnimation` on the `maximumHeight` property. This means adopting superqt and adopting QPropertyAnimation for camera transitions are complementary — the same animation framework is already exercised by QCollapsible. Cite this when justifying the "no new animation dep" argument for camera-transition interpolation.
- **qt-material has only 72 GitHub stars vs pyqtdarktheme-fork's ~746 and its last release was April 2025 (2026-05-21):** When choosing between the two theming engines, the maintenance signal strongly favors pyqtdarktheme-fork. qt-material is not a dead project but its community adoption is an order of magnitude smaller. Always check star count and last-release date before recommending a theming library as a core dep.
