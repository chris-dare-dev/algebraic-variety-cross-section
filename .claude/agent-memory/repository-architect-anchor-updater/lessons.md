- MOVES.md format adjustment: MOVES.md was created correctly by the implementer (Op 11 per PLAN.md §2 MEDIUM-3) before this agent ran. The anchor-updater's role for Batch 4 is to VERIFY the file exists and has 4 entries, not to CREATE it. Always check first.
- verify-anchors.py needle substring issue: the needle `parameter_grid_panel` is a strict substring of the correct new path `panels/parameter_grid_panel.py`. The script will always report false positives for this module's panel file because the name was preserved (not renamed) during the move (PLAN.md v2 MEDIUM-2). When `to` path contains `from` as substring, the "stale anchors found" count will always include false positives. Flag this as a pipeline improvement item; do not attempt to edit correct content to remove the substring.
- CORRECTION block placement: CORRECTION blocks intentionally contain old→new path mappings, which cause the verify-anchors.py count to INCREASE after corrections are appended. This is expected and correct. Final count increase of ~60 (9 CORRECTION blocks × ~7 lines each) from pre-run count is the correct behavior.
- Active restructure notes as historical: The verify-anchors.py `is_acceptable_match` function exempts COMPLETED restructures but NOT the active restructure's own design/audit notes. These notes (PLAN.md, audit briefs, adversary critique) describe the pre-move state and should NOT be edited. Surface this limitation in each batch-4-level report so the orchestrator can decide whether to update the script.
