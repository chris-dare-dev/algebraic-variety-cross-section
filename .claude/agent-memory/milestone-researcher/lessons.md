## enriques-backface-2026q2-e1 (2026-05-22)

- PyVista `culling=True` is equivalent to `culling='back'` — confirmed via `actor.prop.culling == 'back'` check after `add_mesh(mesh, culling=True)`. Default is `culling=None` which maps to `actor.prop.culling == 'none'`.

## realtime-variety-render-e4 (2026-05-22)
- For Qt-threading-refactor milestones, the predecessor SPIKE script (`spike-thread-test.py`) IS the production blueprint — read its `WorkerSignals`/`MeshWorker` classes and recommend porting them verbatim rather than re-deriving the QRunnable pattern. The spike already passed 3/3 stress variants; don't relitigate.
- The e1 queue-latest guard (`_computing` / `_pending_render`) does NOT need replacing for the worker design — it stays as-is; its semantics just *flip* from "blocking compute window" to "worker in flight." The catch-up `QTimer.singleShot(0)` MOVES from the dispatch function's `finally` into the worker-result slot's `finally`, because "in flight" now ends when the signal arrives, not when dispatch returns.
- Two distinct refs are needed for VTK #18782 in an interactive app (not just the spike's one): (1) worker-side `self._result = mesh` before emit, AND (2) `MainWindow.self._active_worker` holding the worker for its whole flight — a local var in the dispatch function is GC'd the instant dispatch returns, killing the worker + its `_result` mid-run.

The slot must drop any result whose `generation_id != self._generation`. Ship warning text + error kind/msg + `perf_counter` duration as fields of the result payload.
- Removing `processEvents()` for a worker design is AI-9-POSITIVE: it deletes the only synchronous event-queue drain in the render path. The new (benign) re-entrancy surface is the QueuedConnection slot, which is serialized with all GUI events and cannot re-enter itself.
