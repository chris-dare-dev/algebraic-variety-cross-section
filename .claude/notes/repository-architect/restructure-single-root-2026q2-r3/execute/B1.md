### Change 2 — symbol_renames data structure (L237-248)

```python
# Before (bug): one entry per source module — last symbol wins
symbol_renames: dict[str, tuple[str, str]] = {}
symbol_renames[old_module] = (new_module, symbol)  # OVERWRITES previous

# After (fix): one entry per (source_module, symbol_name) pair
symbol_renames: dict[tuple[str, str], str] = {}
symbol_renames[(old_module, symbol)] = new_module   # unique key
```

This was the root cause of the B4 multi-destination bug: 45 B4 symbol-map
entries all have `"from": "surfaces"`, so the old dict would have contained
only the LAST entry (`("surfaces", "_hanson_cross_section"): "varieties._marching"`).

### Change 3 — leave_SimpleStatementLine + FlattenSentinel (L103-160)

Replaced `leave_ImportFrom` with `leave_SimpleStatementLine`. The
`leave_ImportFrom` visitor cannot return multiple nodes (it replaces one
ImportFrom with one ImportFrom). The correct hook for splitting one statement
into many is `leave_SimpleStatementLine`, which can return a `FlattenSentinel`.

Multi-alias split logic (pattern C):
1. Extract source module from `stmt.module`.
2. Compute `relevant = {sym: dest for (src, sym), dest in symbol_renames if src == mod_path}`.
3. Group `ImportAlias` objects by their destination module.
4. If all aliases go to the same destination: simple module rename.
5. If aliases go to different destinations: `FlattenSentinel([stmt1, stmt2, ...])`.
6. Aliases not in the symbol-map stay on the original source module (safety).

### Change 4 — leave_Attribute symbol-rename branch (L176-184)

Extended the existing `leave_Attribute` method to handle `kind=symbol` entries.
The old code only handled `module_renames`; `surfaces.ParamSpec` was NOT rewritten.

```python
# New branch after module_renames loop:
dot_idx = chain.find(".")
src_mod = chain[:dot_idx]
rest = chain[dot_idx + 1:]
sym_name = rest.split(".")[0]
dest = symbol_renames.get((src_mod, sym_name))
if dest is not None:
    return _dotted_to_attribute(cst_mod, dest + "." + rest)
```

### Change 5 — METADATA_DEPENDENCIES + MetadataWrapper (L91, L267-270)
