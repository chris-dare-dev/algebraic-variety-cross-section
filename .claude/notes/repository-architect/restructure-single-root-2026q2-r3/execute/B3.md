### Deviation: test_icons.py needed fixes

`tests/test_icons.py` had two categories of stale references to the deleted `icons.py` shim:

1. `importlib.import_module("icons")` (lines 47, 172) — fixed to `"_qt.icons"`
2. `sys.modules.pop("icons", None)` (lines 46, 172) — fixed to `"_qt.icons"`
3. `patch.object(icons, "_qta", mock_qta)` (4 locations) — fixed to
   `patch("_qt.icons._qta", mock_qta)` (string-form prevents stale-module-object
   trap after sys.modules.pop)
4. Module-level `import _qt.icons as icons` alias — simplified to `import _qt.icons`
5. `getattr(icons, color_helper_name)(theme)` — updated to `getattr(_qt.icons, ...)`

Root cause: `test_icons.py` had been partially migrated to `_qt.icons` paths (the
module-level import was already `import _qt.icons as icons`) but still used
`importlib.import_module("icons")` for the lazy-load sentinel test, and still used
`patch.object(icons, ...)` (module-object form) which broke when the first test
did `sys.modules.pop("_qt.icons", None)` causing subsequent patch calls to target
the stale module object.

This is a legitimate B3 fix (the shim deletion unmasked the stale test path); it is
not a scope deviation.
